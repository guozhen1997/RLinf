GR00T-N1.6模型强化学习训练
==================================================================

.. |huggingface| image:: /_static/svg/hf-logo.svg
   :width: 16px
   :height: 16px
   :class: inline-icon

本示例提供了一份完整指南，介绍如何在LIBERO环境中使用RLinf框架，通过强化学习对GR00T-N1.6算法进行微调。内容涵盖从环境设置、核心算法设计到训练配置、评估和可视化的全过程，并提供可复现的命令和配置片段。

主要目标是开发一个能够执行机器人操作的模型，具体包括：

1. **视觉与语言理解**：利用大规模预训练的冻结主干（Backbone）处理多视角 RGB 图像与自然语言任务指令。
2. **平滑连续控制**：基于流匹配（Flow Matching / Diffusion）架构的动作头，生成高频、平滑的连续动作轨迹。
3. **强化学习微调**：通过 PPO 算法结合环境反馈优化策略。

环境
-----------

环境
-----------

**LIBERO环境**

- **环境**：基于robosuite（MuJoCo）构建的LIBERO仿真基准。
- **任务**：控制7自由度机械臂执行各种家庭操作技能（拾取放置、堆叠、打开抽屉、空间重排）。
- **观测**：由放置在工作区周围的离屏摄像头捕获的RGB图像（典型分辨率为128×128、224×224或256×256）。
- **动作空间**：环境原生提供7维连续动作——3D末端执行器位置控制（x、y、z）、3D旋转控制（横滚、俯仰、偏航）、夹爪控制（打开/关闭）。*注：GR00T-N1.6 在底层会通过具身标签将该 7 维动作统一零填充（Zero-padding）至 128 维的跨具身通用动作空间中。*

**任务描述格式**

GR00T-N1.6 直接将环境提供的自然语言任务描述作为语言模型的输入，利用冻结的主干网络提取语义特征。

**数据结构**

- **图像**：主视角和手腕视角的连续 RGB 视频帧，通常命名为“main_images”和“wrist_images”。考虑到时间步历史，形状通常为 ``[batch_size, seq_len, 224, 224, 3]``（具体键名与分辨率受 ``modality_config`` 动态接管）。
- **状态**：末端执行器的位置、姿态和夹爪状态（在网络底层与视觉特征拼接作为状态表征）。
- **任务描述**：自然语言指令。
- **奖励**：用于 PPO 强化的稀疏奖励（成功为 1，失败为 0）

算法
---------

**核心算法组件**

1. **PPO（Proximal Policy Optimization）**

   - 使用GAE（广义优势估计）进行优势估计
   - 带比例限制的策略裁剪
   - 价值函数裁剪
   - 熵正则化



环境与跨具身支持 (Cross-Embodiment)
--------------------------------------

**灵活的具身生态与 Modality Config**

GR00T-N1.6 采用了高度解耦的数据接口设计。通过配置不同的 ``Embodiment Tag`` 与模态转换器，框架不仅原生支持如 LIBERO 中的 7 自由度机械臂（3D位置、3D旋转、夹爪），更能通过同一套底层连续控制参数无缝扩展至具有四条腿的四足机器人形态，无需对底层网络架构做任何侵入性修改。

GR00T-N1.6 核心架构升级
--------------------------------------

**1. FSDP (Fully Sharded Data Parallel) Actor Worker**

鉴于 GR00T-N1.6 参数规模的显著增长，RLinf 的 Actor 节点现已全面重构为 ``EmbodiedFSDPActor``。该架构将模型权重、梯度与优化器状态跨多 GPU 节点分片，打破了传统 DDP 的单卡显存瓶颈，极大提升了吞吐量。

**2. Head-only 微调与防爆机制 (Gradient Protection)**

为适应少量数据的物理环境对齐并节省计算资源，GR00T-N1.6 的 RL 训练采用部分微调策略：
- **主干冻结 (Backbone Freezing)**：严格锁定数以十亿计的视觉和语言主干网络参数，使其仅以推理模式（Eval）运行。
- **动作头精细控制**：仅解冻动作输出头（Action Head）以学习物理动力学。更进一步，框架对时间步编码器（``timestep_encoder``）进行了强制冻结与梯度 Hook 拦截，精准防范了高维动作分布中极易引发的 ``NaN`` 梯度爆炸问题。

GR00T-N1.6 的预备知识
-----------------------------
此处介绍 GR00T-N1.6 的核心架构升级与重要设计理念，以帮助开发者更深入地理解底层逻辑并高效部署该模型。

**1. 两阶段解耦训练范式 (Two-Stage Training Pipeline)**

RLinf 框架针对 GR00T-N1.6 采用了高度解耦的两阶段训练架构，SFT 与 PPO 在逻辑上完全物理隔离：
- **第一阶段（纯 SFT 预热）**：采用 ``Pure SFT Model`` 模式。模型完全脱离物理仿真环境，仅依赖离线专家数据集（如 LeRobot/LIBERO）进行监督微调，专注拟合目标动作轨迹。此阶段会屏蔽所有环境 Rollout 与采样接口。
- **第二阶段（PPO 强化对齐）**：在 SFT 收敛的基础上，将模型载入基于 FSDP 的分布式 Actor 中，与仿真环境进行实时交互，通过优势函数（Advantage）和 PPO 算法进行物理世界约束和闭环微调。

**2. 极简的局部微调策略 (Head-only Fine-Tuning)**

为了在大幅节省显存的同时防止“灾难性遗忘”（即丧失预训练模型原有的视觉语义理解能力），框架默认采用“冻结主干”策略：
- **主干冻结 (Backbone Freezing)**：在 SFT 和后续 RL 阶段中，庞大的视觉-语言主干网络将被严格锁定（``requires_grad=False``），仅作为特征提取器。
- **专注动作头**：仅解冻动作输出头（Action Head）参与梯度更新，极大降低了显存开销，使得在常规算力集群上进行数十亿参数大模型的微调成为可能。

**3. 流匹配动作生成 (Flow-Matching Action Head)**

与将动作强制离散化为 Token 的传统自回归范式不同，GR00T-N1.6 采用了前沿的连续动作生成方案。
- 模型通过加噪与去噪的流匹配机制（Flow-SDE / Diffusion），直接在连续空间中生成高频、平滑的未来动作块（Action Chunking）。
- 关键配置：通过 ``num_action_chunks`` 控制单次推理预测的未来时间步长，``denoising_steps`` 控制去噪采样的深度。

**4. 高度泛化的模态配置 (Modality Config & Cross-Embodiment)**

通过高度抽象的数据接口设计，模型彻底解耦了具体硬件形态。
- **具身标签 (Embodiment Tag)**：依靠传入的配置标签（如 ``ROBOCASA_PANDA_OMRON``），系统能动态适配对应的状态编码器与动作空间。无论是单臂机械臂，还是拥有四条腿的四足机器人等其他形态，底层连续控制逻辑均可复用，实现跨具身泛化。

**5. FSDP 分布式并行架构 (Fully Sharded Data Parallel)**

面对 GR00T-N1.6 庞大的参数规模（如 3B 模型），传统的分布式训练在环境采样和反向传播交替时极易遇到显存瓶颈。
- 底层系统针对 Actor 节点进行了重构（如 ``EmbodiedFSDPActor``），能够跨 GPU 节点对模型权重、梯度与优化器状态进行分片切分（Sharding）。这极大突破了单卡显存天花板，保证了在大规模高维动作空间训练时的高效吞吐。

运行脚本
---------------

**1. 关键集群配置**

.. code:: yaml

   cluster:
      num_nodes: 1
      hardware_ranks: [[0]]
      component_placement:
        actor,env,rollout: all

在此处，您可以灵活配置env、rollout和actor组件的GPU数量。

--------------

**2. 模型关键参数配置**

GR00T-N1.6 的 PPO 配置极大地精细化了对模型分布式切片与动作生成的控制。您可以参考配置代码中的核心模块进行调优：

**2.1 Actor 模型与动作头配置 (`actor.model`)**

.. code-block:: yaml

   model:
     model_type: "gr00t_1_6"   
     add_value_head: True          # 强化学习关键：动态注入价值网络预测优势
     num_action_chunks: 1          # PPO阶段调为1，提升高频物理交互的实时性
     denoising_steps: 1            # 控制流匹配(Flow-Matching)去噪步数

- **动作分块与去噪 (`num_action_chunks`, `denoising_steps`)**：
  在 PPO 闭环训练中，为了加快环境 Rollout 的响应频率并减少计算开销，建议将这二者调低（例如配置中的 ``1``）。
- **价值网络注入 (`add_value_head: True`)**：
  框架支持在 GR00T 内部动态挂载 Critic 价值头，与 Actor 共享主干特征提取，以计算精确的 PPO 优势函数（Advantage）。

**2.2 FSDP 分布式切片策略 (`actor.fsdp_config` & `no_split_modules`)**

为了保证 FSDP 能够正确、高效地在多 GPU 间分发 GR00T-1.6 这类多模态复杂架构的参数，必须显式定义哪些核心组件“不可被过度切碎”：

.. code-block:: yaml

   no_split_modules:
     - "Qwen3DecoderLayer"                             # 语言大模型层
     - "Siglip2EncoderLayer"                           # 视觉编码器层
     - "FlowMatchingActionHeadForRLActionPrediction"   # 核心动作预测头
     - "TimestepEncoder" / "TimestepEmbedding"         # 扩散时间步编码
     - "ValueHead" / "ExploreNoiseNet"                 # RL 特有网络

- **策略 (Strategy)**：
  配置中使用了 ``fsdp`` 策略，上述定义的模块将被作为 FSDP 封装的最小原子单元（Transformer Wrapper），防止因跨层切割导致前向/反向传播时的严重通信开销（Communication Overhead）。

**2.3 PPO 与优化器超参数 (`algorithm` & `actor.optim`)**

.. code-block:: yaml

   algorithm:
     adv_type: gae                 # 广义优势估计
     clip_ratio_high: 0.2          # 策略裁剪上限，防止单次更新过度破坏策略
     clip_ratio_low: 0.2           # 策略裁剪下限
     gamma: 0.99                   # 折扣因子
     gae_lambda: 0.95              # GAE 优势平滑权重

   optim:
     lr: 5.0e-6                    # Actor (策略网络) 学习率
     value_lr: 1.0e-4              # Critic (价值网络) 学习率通常设置得比 Actor 大
     clip_grad: 1.0                # 梯度裁剪阈值，配合防爆机制进一步稳定训练

**2. 关键监控指标**

- **训练指标**

  - ``actor/loss``：策略损失
  - ``actor/value_loss``：价值函数损失（PPO）
  - ``actor/grad_norm``：梯度范数
  - ``actor/approx_kl``：新旧策略之间的KL散度
  - ``actor/pg_clipfrac``：策略裁剪比例
  - ``actor/value_clip_ratio``：价值损失裁剪比例（PPO）

- **rollout指标**

  - ``rollout/returns_mean``：平均回合回报
  - ``rollout/advantages_mean``：平均优势值

- **环境指标**

  - ``env/episode_len``：平均回合长度
  - ``env/success_once``：任务成功率
